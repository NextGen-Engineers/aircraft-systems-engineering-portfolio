# Requirements Engineering

## Overview

Requirements engineering is the process of defining what a system must do before designing or implementing it.

This project demonstrates how stakeholder needs are translated into system requirements and then into software requirements for a simplified commercial aircraft fuel management system.

---

## Objectives

- Understand stakeholder needs
- Develop system requirements
- Develop software requirements
- Maintain traceability
- Prepare for software design and verification

---

## Project Scope

System:

Commercial Aircraft Fuel Management System (Educational Simulation)

Purpose:

Monitor aircraft fuel levels, operate fuel pumps, monitor abnormal conditions, and provide information to the flight crew.

---

## Deliverables

- Stakeholder Needs
- System Requirements
- Software Requirements
- Requirement Traceability Matrix
- Requirement Verification Plan
